# /search-first — 🔎 搜尋優先開發模式 (Research Before Coding)

系統化地執行「在實作前先尋找現有解決方案」的工作流。

## 觸發情境

當執行以下任務時使用此技能：
- 開始一個可能已有現成解決方案的新功能。
- 添加新的依賴 (Dependency) 或整合項目。
- 用戶要求「增加 X 功能」，而你正準備動手寫程式碼。
- 在建立新的工具程式 (Utility)、輔助工具 (Helper) 或抽象層 (Abstraction) 之前。

## 工作流 (Workflow)

```
┌─────────────────────────────────────────────┐
│  1. 需求分析 (Need Analysis)                   │
│     定義所需的具體功能                          │
│     識別語言 / 框架限制                        │
├─────────────────────────────────────────────┤
│  2. 並行搜尋 (Parallel Search)                │
│     ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│     │  npm /   │ │  MCP /   │ │  GitHub / │  │
│     │  PyPI    │ │  Skills  │ │  Web      │  │
│     └──────────┘ └──────────┘ └──────────┘  │
├─────────────────────────────────────────────┤
│  3. 評估 (Evaluate)                          │
│     為候選方案評分 (功能、維護狀況、            │
│     社群、文件、授權、依賴項)                  │
├─────────────────────────────────────────────┤
│  4. 決策 (Decide)                            │
│     ┌─────────┐  ┌──────────┐  ┌─────────┐  │
│     │ 直接採用 │  │  擴充     │  │  客製化  │  │
│     │ (Adopt) │  │  / 封裝   │  │  建置    │  │
│     └─────────┘  └──────────┘  └─────────┘  │
├─────────────────────────────────────────────┤
│  5. 實作 (Implement)                         │
│     安裝套件 / 配置 MCP /                      │
│     撰寫最少量的客製化代碼                      │
└─────────────────────────────────────────────┘
```

## 決策矩陣 (Decision Matrix)

| 訊號 (Signal) | 建議行動 |
|---|---|
| 完全匹配、維護良好、MIT / Apache 授權 | **直接採用 (Adopt)** — 直接安裝並使用 |
| 部分匹配、具備良好基礎 | **擴充 (Extend)** — 安裝並撰寫精簡封裝 (Wrapper) |
| 多個弱匹配方案 | **組合 (Compose)** — 組合 2-3 個小型套件 |
| 找不到合適的方案 | **建置 (Build)** — 根據研究結果進行客製化開發 |

## 如何使用

### 快速模式 (Inline)

在撰寫工具程式或添加功能前，請在腦中快速掃描：

0. 專案庫中是否已存在？ -> 先用 `rg` 搜尋相關模組或測試。
1. 這是一個常見問題嗎？ -> 搜尋 npm / PyPI。
2. 是否有現成的 MCP？ -> 檢查配置並搜尋。
3. 是否有現成的專屬技能 (Skill)？ -> 檢查 `.agent/skills/` 目錄。
4. 是否有 GitHub 的參考實作或模板？ -> 在撰寫全新代碼前先進行 GitHub 代碼搜尋。

### 完整模式 (Agent)

對於非瑣碎的功能，啟動研究亞代理：

```python
Task(subagent_type="general-purpose", prompt="
  研究現有工具以實現：[功能描述]
  語言 / 框架：[語言]
  限制條件：[任何限制]

  搜尋範圍：npm/PyPI, MCP 伺服器, Claude Code 技能, GitHub
  回傳：帶有建議的結構化比較報告
")
```

## 整合點 (Integration Points)

### 與任務調度中心 (PM) 聯動
PM 在架構評核階段前應先調用研究員：
- 研究員識別可用的現成工具。
- 規劃者將其納入實作計畫中，避免無謂的工作。

### 與架構分析聯動
在進行以下決策時諮詢研究員：
- 技術棧決策。
- 整合模式的發現。
- 現有參考架構的調閱。

## 反模式 (Anti-Patterns)

- **直接跳入編碼**：在沒檢查是否有現成解決方案前直接開寫。
- **忽視 MCP**：未確認 MCP 伺服器是否已提供此能力。
- **過度客製化**：對函式庫進行過重封裝，失去了原有的優點。
- **依賴膨脹**：為了實現一小項功能而安裝龐大的套件。
