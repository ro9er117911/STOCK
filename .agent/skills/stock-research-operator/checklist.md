# 研究品質驗收清單 — Pre-Delivery Checklist

> 每次交付研究前，逐條檢查。未通過項目必須補齊後才可交付。

---

## A. 時間軸檢查 ⏱️

- [ ] 明確定義持有期（具體寫出週/月/年）
- [ ] 觀察變數與持有期匹配（短線看 newsflow，長線看結構）
- [ ] 設定降噪規則（哪些資訊依時間軸應被忽略）
- [ ] 設定警鈴閾值（什麼程度的變化才需要動作）

---

## B. Thesis 檢查 🎯

- [ ] 是一句可驗證的主張（非泛泛之談）
- [ ] 包含因果主線
- [ ] 包含明確催化事件
- [ ] 包含時間範圍
- [ ] 包含失效條件（什麼情況下 thesis 作廢）

---

## C. 假設檢查 🧪

- [ ] 至少拆出 3 個核心假設
- [ ] 每個假設都可觀察、可驗證
- [ ] 每個假設都有失效條件
- [ ] 每個假設都連結到股價驅動因素
- [ ] 沒有出現「公司很好」「產業有前景」等模糊假設

---

## D. 風險檢查 ⚠️

- [ ] 至少列出 3 個可能推翻 thesis 的風險
- [ ] 不只有總經/黑天鵝風險（含結構、執行、競爭、需求、監管）
- [ ] 每個風險有先行訊號（什麼指標會先動）
- [ ] 每個風險有對應動作（觀察/減碼/撤退）
- [ ] 已標記 thesis breaker vs. thesis weakener vs. noise

---

## E. 情境檢查 🔀

- [ ] 情境分析包含因果邏輯切換（非只改數字）
- [ ] 至少有 Base / Bull / Bear / Break 四種情境
- [ ] 每個情境有觸發條件
- [ ] 每個情境有變化鏈條與利潤重新分配邏輯
- [ ] 每個情境對應具體動作
- [ ] Pre-mortem 已完成（假設虧損 50% 的倒推分析）

---

## F. 比較檢查 🔍

- [ ] 與 PLTR / MSFT / MAR 或同主題標的做橫向比較
- [ ] 明確回答「為什麼選它不是別家」
- [ ] 評估上行彈性（主假設成立時誰最受益）
- [ ] 評估下行保護（主假設失敗時誰跌最少）
- [ ] 考慮資產負債表韌性

---

## G. 行動檢查 ✅

- [ ] 清楚寫出「現在該做什麼」
- [ ] 清楚寫出「錯了怎麼辦」
- [ ] 有明確加碼條件
- [ ] 有明確減碼條件
- [ ] 有明確撤退條件
- [ ] 有明確換股條件
- [ ] 設定下次更新日期與必追資料

---

## 最終確認 🏁

| # | 成功標準 | 通過？ |
|---|---|---|
| 1 | 讀者 3 分鐘內知道看多/看空什麼、為什麼、何時驗證 | ☐ |
| 2 | 每個結論都能追溯到可驗證假設 | ☐ |
| 3 | 新資訊進來時，能快速分流：忽略/觀察/加碼/減碼/撤退 | ☐ |
| 4 | 清楚回答「為什麼買它，不是別家」 | ☐ |
| 5 | Thesis 失效時能快速切換，而非事後合理化 | ☐ |

---

> ❌ 任何一項「最終確認」未通過 → 研究不可交付，回頭補齊。
