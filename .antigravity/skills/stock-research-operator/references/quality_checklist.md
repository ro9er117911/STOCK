# 雙重品質檢核表 (Quality Checklist)

此檢核表分為兩個部分：
1. **研究品質驗收（給 Agent 或使用者在產出內容後確認）**
2. **Skill Readiness Gate（給 Skill Creator，確認本 Skill 架構是否吻合標準）**

---

## Part 1. 研究產出品質檢核 (Pre-Delivery QA Gate)

> 每次交付研究結果（尤其是 `assets/template.md`）之前，必須確認是否通過下列標準。任一不可預期錯誤即為產出不合格。

### A. 邏輯與條件 ⏱️🎯
- [ ] **持有期與 Thesis：** 必須包含具體持有期（週/月/年）以及一句完整的包含因果鏈與催化劑的主張。
- [ ] **變數聚焦：** 核心與次要變數必須與持有期匹配，並且有定義明確的降噪規則。
- [ ] **假設與可驗證性：** 至少三個假設，每個假設皆「可觀察、可驗證、並附帶失效條件」。

### B. 風險與情境 ⚠️🔀
- [ ] **風險多樣性：** 除了大環境風險，必須有商業模式/競爭/執行上的特定風險。
- [ ] **風險層級分級：** 明確標記 thesis breaker (撤退), thesis weakener (減碼), noise (無視)。
- [ ] **情境行動化：** 所有 Base / Bull / Bear / Break 皆帶有明確的行動指標，不只是數字改動。
- [ ] **Pre-mortem 分析：** 預演 50% 虧損時最可能的盲區死穴。

### C. 決策導向 🔍✅
- [ ] **同行比較：** 包含至少兩名同業，並回答「為何選他不選別人？」。
- [ ] **明確的執行門檻：** 現在要做什麼、何時加碼、何時減碼、何時撤出，定義完整。

### 最終放行標準
1. 讀者 3 分鐘內能判讀：看多或空？為什麼？何時驗證？
2. 新資訊進來能依照閾值分流：忽略/觀察/加碼/減碼/撤退。
3. 結論均能追溯到可被檢驗的單一指標上。

---

## Part 2. Skill Readiness Check (Skill Creator Audit)

> 若要對本 Skill 進行更新推送，請確保本 Skill 符合 `skill-creator-advanced` 的標準。

- [ ] **邊界與定位清楚：** `<decision_boundary>` 已明定，能清楚切分與其他分析 skill 的不同。
- [ ] **單一職責：** 該 skill 不混淆其他與股權研究無關的行政工作。
- [ ] **Output Contract 穩定：** 對長格式輸出的依賴已被下放到 `assets/template.md`。
- [ ] **工作流明確：** `<workflow>` 採用祈使句，且每一步有明確 I/O，不依賴 LLM 空泛想像。
- [ ] **Policy 定義完善：** `<default_follow_through_policy>` 定義了 Agent 能主動調用哪些資訊，以及何時必須等待使用者。
- [ ] **Format 與 Metadata：** `SKILL.md` 的 YAML 區域無誤，工具與路徑均正常存在。
