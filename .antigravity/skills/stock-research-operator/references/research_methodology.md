# 投資研究核心方法論

> Agent 執行研究或修正時的參考指導原則。

## 1. 核心心法

### 1.1 時間軸優先 (Time-Horizon First)
在進行任何分析前，必須先定義持有期（週/月/年）。時間軸決定了：
- 哪些變數最重要
- 資訊權重如何分配
- 什麼事件屬於噪音
- 何種事件會觸發動作

**時間軸與變數對應表：**
- **短線 (數週)**：新聞流、事件催化劑、預期落差、籌碼擁擠度。
- **中線 (數季)**：訂單、庫存、毛利趨勢、共識獲利調整。
- **長線 (數年)**：市佔率、產業結構、資本配置、護城河。

### 1.2 邊際變化大於資訊量
價值不在於列出所有資料，而在於回答：
- 該事件是否改變 Thesis？
- 改變程度多大？
- 是否突破警鈴閾值？

### 1.3 情境必須導向具體行動
每一個推論都必須轉化為條件式反應：
- 如果 A 發生 → 執行 X 動作
- 如果 B 發生 → 執行 Y 動作
- 如果 C 發生 → 撤出或換股

### 1.4 評價體系重於單一目標價 (Valuation Regime > Point Estimate)
不要直接給出單一估值或 PE 數字，這會忽略市場偏好轉換。必須思考：
- 現在市場用什麼指標幫它定價？（例如 P/B、P/E、EV/EBITDA）
- 有可能發生體系切換嗎？從價值股重估為成長股的觸發點？
- 潛在的風險是什麼？

### 1.5 迭代型校準
好的研究不是一次性產出。每次有新事件時，必須重新確認核心假設是否仍成立、風險是否實現、行動規則是否需要微調。

## 2. 訊號與噪音判別規則 (Signal vs. Noise)

**🟢 應視為核心訊號 (Signal)：**
- 多個指標同時往同一方向變動。
- 變化直接擊中核心假設。
- 具備持續性，而非一次性事件。
- 有改變評價體系 (Valuation Regime) 的潛力。
- 改變了產業利潤的重新分配。

**🔴 應視為噪音過濾 (Noise)：**
- 單一月份、單一來源、或尚未經證實的傳聞。
- 僅影響市場情緒，但根本面（Fundamentals）未變。
- 無法與 Thesis 因果鏈產生連結。
- 變動幅度低於設定的「警鈴閾值」。

## 3. 常見錯誤模式 (Failure Modes)

1. **未定義時間軸：** 分析時沒有先回答這是持有三週還是三年的操作。
2. **資料拼盤未去蕪存菁：** 貪心地混用大盤新聞、法說會、傳聞，卻沒有照時間軸賦予權重。
3. **沒有情境與動作關聯：** 列出華麗的情境預測，卻沒有寫「如果變成這樣，我該如何動作」。
4. **執著點位估值：** 對一個估值邏輯（DCF、PE倍數）斤斤計較，卻沒發現市場正在換另一把尺（Regime shift）。
5. **單獨看待無比較對象：** 研究公司 A 時未列出競爭對手 B，導致無法回答「為何不買更便宜/更安全的同行」。
6. **未準備 Pre-mortem：** 只會做上行樂觀預測，沒有先預演「如果大賠 50%，最可能是哪個前提失敗了」。
7. **情緒性修改 Thesis：** 新聞一出來就急著改結論，而不是依照預設閾值進行驗證分流。

## 4. 事件更新流程 (Event Update Protocol)

當有新資訊或季報發佈時，不要急著改寫 `template.md`，按照以下分流邏輯處理：
1. 資訊時效是否符合原本定義的持有期？
2. 它擊中了哪個「核心假設」？
3. 對該假設的邊際貢獻是正向 (+)、負向 (−) 還是無影響 (0)？
4. 是否突破原本設定的警鈴閾值？
5. 最終決定動作：忽略、維持、加碼、減碼、撤出。
